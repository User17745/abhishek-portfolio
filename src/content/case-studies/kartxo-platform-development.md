## Project Overview

GreenHonchos identified a gap in the market for SMB-focused D2C platforms that understood Indian consumer behavior. KartXO was born from this insight - a platform that would enable small to medium brands to launch professional eCommerce without enterprise costs.

## The Challenge

The Indian SMB market had specific needs that existing platforms didn't address:

1. **OTP Login**: Indian consumers prefer OTP over email/password
2. **Local Payments**: UPI, COD, and regional wallets are essential
3. **Easy Customization**: Non-technical founders need DIY tools
4. **Affordable Pricing**: SMB-friendly pricing models

## My Role

As the sole developer for the first version, I wore multiple hats:

- **Product Manager**: Feature prioritization and roadmap
- **Backend Developer**: API design and database architecture
- **Frontend Developer**: Store templates and admin panel
- **DevOps**: CI/CD and deployment pipelines

## Technical Architecture

### Backend
- Node.js with Express framework
- MongoDB for flexible schema needs
- Redis for caching and sessions
- AWS infrastructure with auto-scaling

### Frontend
- React-based admin panel
- Server-side rendered store templates
- Progressive Web App capabilities
- Mobile-first responsive design

### Key Features Built

**Authentication**
- OTP-based login via SMS
- Social login (Google, Facebook)
- Guest checkout with phone capture

**Store Management**
- Drag-and-drop page builder
- Product catalog with variants
- Inventory management
- Order processing workflow

**Payments & Logistics**
- Integrated with Razorpay and Paytm
- Multi-carrier shipping calculation
- COD support with verification
